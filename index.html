👥 SocialDS Data Structures Project — MCA Final Year

Social Media Feed & Friend Management

Demonstrating Hash Table • Graph • Queue • Stack • File Handling • CGI in C

📈 Hash Table 🔗 Graph 📰 Queue ↻ Stack
👤

Create User

Registers a new user. Username is stored in a Hash Table using the djb2 hash function with separate-chaining collision handling.

Letters, digits, underscores only (max 32 chars)
📝

Add Post

Publishes a new post, enqueued at the REAR of the FIFO Queue. Feed shows posts in chronological order from the FRONT.

0 / 480 characters
🔗

Add Friend

Creates a mutual friendship — an undirected edge in the Graph (adjacency list representation). Both users see each other's posts.

📰

View Feed

Displays posts from the user and their friends. The Queue is traversed FIFO; the Graph filters which posts are visible.

↻

Undo Last Post

Uses a Stack (linked list) to find and remove the user's most recent post — classic LIFO undo pattern.

📊

All Users (Hash Table)

Displays every registered user with their hash bucket index — useful for understanding how the hash table distributes keys.

📚 Data Structures Used

Friend Network Graph (Adj. List) Model mutual relationships as undirected edges Add edge: O(1) • Check: O(degree)
Post Feed Queue (FIFO) Chronological display, oldest post first Enqueue / Dequeue: O(1)
